## Part 1: Greenfield Implementation (Coding)

You must implement a system to track different types of events (movement, combat) generated by robots.

### Step 1 — Implement `TelemetryEvent` (Abstract Class)
This is the abstract base class for all telemetry data.
* **Fields:** `robotId` (String), `timestamp` (long). Both must be `private` and immutable after creation.
* **Constructor:** Initializes `robotId` and `timestamp`.
* **Methods:** Provide getters for both fields. Declare an abstract method: `public abstract String getEventSummary();`

### Step 2 — Implement Concrete Subclasses
Create two classes that extend `TelemetryEvent`.

**1. `MovementEvent`**
* **Extra Fields:** `x` (int), `y` (int) — representing the coordinates moved to.
* **Constructor:** Accepts `robotId`, `timestamp`, `x`, and `y`.
* **Summary:** `getEventSummary()` must return: `"Moved to [x, y]"`

**2. `CombatEvent`**
* **Extra Fields:** `damageDealt` (int). Must not be negative.
* **Constructor:** Accepts `robotId`, `timestamp`, and `damageDealt`. Throws an `IllegalArgumentException` if damage is less than 0.
* **Summary:** `getEventSummary()` must return: `"Engaged in combat, dealt [damageDealt] damage"`

### Step 3 — Implement `TelemetryTracker`
This class acts as the central hub for storing the data. It must use a `Map` to associate a robot's ID with a `List` of its events.
* **Fields:** A private `Map<String, List<TelemetryEvent>>` named `eventLog`.
* **Constructor:** Initializes the map.
* **Methods:**
    * `public void recordEvent(TelemetryEvent event)`: Adds the event to the map under the correct `robotId`. If the robot does not exist in the map yet, create a new list for it.
    * `public List<TelemetryEvent> getRobotEvents(String robotId)`: Returns a list of events for the specified robot. If the robot has no events, return an empty list.
    * `public int getTotalDamageDealt(String robotId)`: Loops through the robot's events. If an event is a `CombatEvent`, add its damage to a running total. Return the total.
